Write an RFC for migrating our monolithic Django backend to an event-driven architecture using message queues. Currently all inter-module communication is synchronous function calls within a single Django process. We have three bounded contexts that are tightly coupled: billing, notifications, and user management. The billing module directly calls notification functions to send receipts, and user management triggers billing recalculations on plan changes. We want to decouple these with async events so each can be deployed and scaled independently. This targets both C4 (new containers for message broker and separate services) and ADR (choosing event-driven over synchronous).
