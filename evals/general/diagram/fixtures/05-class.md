Draw a class diagram for a payment processing library. There's an abstract PaymentProvider with abstract methods process(request: PaymentRequest): PaymentResult and refund(transactionId: string, amount: float): RefundResult. Concrete implementations: StripeProvider, PayPalProvider, BraintreeProvider — all extend PaymentProvider. PaymentRequest is a value object with fields: amount (float), currency (string), description (string), metadata (Map). PaymentResult has: transactionId (string), status (enum: success/failed/pending), errorCode (string?). RefundResult has: refundId (string), status (enum), amount (float). PaymentProvider has a private config of type ProviderConfig which has apiKey (string) and sandbox (boolean).
