Draw the binary structure of a custom application protocol header. The header has these fields in order: Version (bits 0-3, 4 bits), Message Type (bits 4-7, 4 bits), Flags (bits 8-15, 8 bits), Total Length (bits 16-31, 16 bits), Correlation ID (bits 32-63, 32 bits), Source Node ID (bits 64-79, 16 bits), Destination Node ID (bits 80-95, 16 bits), Sequence Number (bits 96-127, 32 bits), Timestamp (bits 128-159, 32 bits), Payload Checksum (bits 160-191, 32 bits).
