Create a sequence diagram showing how a REST API handles a file upload. The browser sends a multipart POST to the API Gateway. The gateway authenticates the JWT token by calling the Auth Service. Auth Service returns valid/invalid. If invalid, gateway returns 401. If valid, the gateway streams the file to S3 and gets back a storage URL. The gateway then saves file metadata (name, size, S3 URL, uploader ID) to PostgreSQL. Finally, it publishes a "file.uploaded" event to the message queue, which triggers the Notification Service to send a webhook to the user's callback URL. Gateway returns 200 with the file URL.
